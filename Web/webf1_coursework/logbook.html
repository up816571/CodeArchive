Literature review research

The first task in the website foundations course was to create a literature review relating to personal information exposure on the web. The main focus of my review was on Social engineering and what you can then do with this information. Social engineering is a process which involves gathering data about an individual. More on this can be found in my literature review page.

Spear phishing is a primary means to gain access into someone's personal or business accounts which can be aided by social engineering. This has a large risk to the business or person. There are many ways which social engineering can be used in order to gain access to unauthorized accounts by posing as someone the target may know to try get information.

In the process of researching i also found that simple pieces of information about a person were mainly used as their passwords or security questions. Many companies have policies on their terms of service to try and counteract over sharing information but generally don't have the highest success rate.

The literature review highlighted to be careful with how much information is shared and to be careful with whom this information is shared with. Dr Nick Savage made a point be careful now as your position in the future could be at risk by people you have shared information within the past.

HTML

This is the main content of the webf1 and it started of by looking at hyper text markup language or html for short. HTML is how a web page is marked up and there are a number of tags that can be used. Some main content separation tags are as follow:

  • Header - used to declare all metadata in which describes the other data in the page.
  • Body - where the majority of content will go.
  • Nav - A HTML5 tag that is used to declare links in i.e. the navigation section of the webpage.
  • Footer - used to design the footer or bottom part of the page, normally has a validation link and author of the page in and potential copyright detail.
  • Section - Can be used to define a section of a page.

Metadata tags to define the information about the page

  • Link - used to link other documents such as style sheets.
  • Title - the title of the page which will be shown in the browser

Text separation which are used to separate out content on a page

  • H - Heading tags from 1 to 6 to show headings.
  • Div - Used to group similar content so that it can all be styled as one element.
  • P - used for separating paragraphs.
  • Main - Specifies where the main content is and is used similarly to the nav tag, it was also introduced in HTML5.
  • ol and ul - Order list and unordered list to list a set of things.

Text formatting which is used to affect how text is presented

  • A - The anchor tag which is used to link other web pages or other things such as an email address. The A tag needs to be given a link in the tag defined by href=””.
  • B - The bold tag to highlight certain text
  • Strong - Shows the text is important and defaults to making the text bold.
  • Em - Emphasized text which can also be nested to make the text more emphasized at each level. Explained better at MDN’s em article.
  • I - italic tag which is offset from the rest of the text.
  • U - which is used to underline the text.

Media tags for things like videos and images

Img - Used to show an image on the page, the image will also have an alt tag which is needed for people with a screen reader or other disabilities so they know what the image is of. The image tag also needs scr for the source of the image.

Video - Used to put a video into a web page and also includes a source tag. Most videos will also include a transcript for those that can’t hear or have other disabilities.

Scripting tags showcases in several lectures.

  • Canvas - to designate an area to be a canvas to draw images or animations on.
  • Script - to put a script in such as a java script

Many more element that html uses can be found on the Mozilla development networks element list. This site has been a extremely helpful tool in learning what each element does and how best to use them.

CSS

Is a descriptive markup which helps add meaning the site and its information. CSS stands for cascading style sheet. Having good css can reduce maintenance cost of a site and also reduce costs by only having to send a few css’s to users rather than having styles within the html.

CSS can target specific tags that are used in HTML so they will ll have the same style for example giving the main element a font-family of ariel will make all the text in main have the font arial. Selectors can also be used to target specific content. If in the html a tag was given an ID, it can be used in the css to only style that tag. ID’s are unique so cannot be used again on the same page as an id can be linked to to take the user to a specific place on a page. Giving a class to tag is similar to an Id but the same class can be reused on the page.

The box model is also used on CSS to define the space that the elements take up. First goes the actual continent which will have a width and a height which can be declared in css. Next is padding which is used to define the inner part of the box. After that is a border which by default is set to be nothing. After is the margin which is outside of the box and pushes other elements away like padding will. A useful diagram can be found on MDN’s page.

SCSS is another tool that is being developed. It is similar to CSS but you can declare variables and it has other utility that is not part of regular CSS. At the moment is not nearly as wide used as CSS but s it develops more it could see a much larger use.

Links

Links are used to navigate webpages on line using a URL which stands for uniform resource locater. There are several formats used such as ftp for file transfer. Mailto for email, http/https for web pages and secure web pages. Fragments can be used to go elements with ID’s. Using a / will try go to a subfolder of a webpage to access that page.

Scritpting

Scripts can be used on a webpage to give animations and other features. The scripts we have looked at in WEBF1 are written in javascript. The scripts can make the page more interactive with the user and act dynamically to certain events. A common use of javascript is validation when a form is being filled out.

Accessibility

A web page has to accessible to all users and it is a criminal offense to discriminate against certain groups. The main ways to make a web page accessible is to keep content and form separate with good html and css. Other tags can be used to help certain disabilities like the alt tag for images and a transcript for videos. Aria tags can now also be used to define what objects on html like the main content or a button to help those with disabilities. Making sure a website is not reliant on colour code is also important as some colourblind people might not be able to distinguish if a box is red or green for errors as an example.

SVGs

Scalar vector graphics are a type of image that scale as the size of the image is increased. They are much smaller than standard PNGs and JPEGs as only a few things are defined such as a curve from one point to another the line colour and fill. This reduces the amount of data sent and makes loading a page easier which will result in happy users.

Databases

Databases can be used on webpages to store data. This means that the web page can display only data requested from the database so the page can change more freely in content depending on the user. Using a database is also dependent on have good CSS and HTML as adding new elements from a database would be incredibly time consuming if they needed to be formated every time they were added.
